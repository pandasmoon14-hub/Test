# Astra Paths v0.2 (Cross-Donor Revised)

## Scope of this file

This file is **path-facing**, not a full owner of universal player architecture.

Paths remain one major Astra build layer. They help define how an Ascendant begins, organizes growth, and expresses a recognizable role doctrine. They do **not** own every lawful source of differentiation.

A complete Astra build may also be shaped by other layers, including:
- body or origin structure,
- culture or background structure,
- access channels and permissions,
- social capital, contacts, or starting resources,
- bonded objects,
- augmentations or installed systems,
- interface, resonance, or equivalent donor-local special channels,
- licenses, credentials, or other gated authorities.

This means the existence of strong Paths does not imply path monoculture.

---

## Path doctrine

A Path is a major role-facing growth package. It establishes a coherent opening direction for the character without claiming permanent exclusivity over all later development.

A Path may define or influence:
- recommended Attribute priorities,
- typical Domains or equivalent access keys,
- starting Resistances or defensive tendencies,
- initial codified active expressions,
- ordinary trained capabilities strongly associated with the Path,
- progression hooks,
- aesthetic or narrative manifestation notes.

A Path should answer one central question:

**What kind of Ascendant is this character becoming, and by what doctrine of growth do they begin?**

---

## Paths and other build layers

Astra treats Paths as compatible with stacked build structure.

That means:
- a Path may coexist with strong background pressure,
- a Path may coexist with contacts or social-capital pressure,
- a Path may coexist with augmentation or installed-system pressure,
- a Path may coexist with bonded relic pressure,
- and a Path may coexist with donor-local access states, ratings, or permission channels.

Some donors emphasize Paths heavily. Others distribute build pressure across resources, channels, permissions, licenses, or installed capabilities. Astra must host both.

The stable rule is:

**Path is one lawful owner of build identity, not the sole owner.**

## What this file contains

This file keeps a small set of Astra-valid starter Paths as **exemplars**.

These are not an exhaustive final roster. They remain useful because they provide stable role examples for:
- martial pressure,
- restoration and bodily protection,
- decay and attrition,
- codified ranged or shaped workings.

The current exemplar set is still RHBF-seeded in ancestry, but it is no longer treated as proof that all Astra Path doctrine is cultivation-shaped.

## Path record guidance

A mature Astra Path record should describe, at minimum:
- Path name,
- role identity,
- recommended Attribute priorities,
- typical Domains or equivalent permissions,
- defensive emphasis,
- common capability patterns,
- common active-expression families,
- growth hooks,
- notes on interaction with other build layers where relevant.

A Path record may also specify:
- whether the Path grants starting techniques or only access to them,
- whether the Path assumes ordinary skill training, codified active expressions, or both,
- whether the Path interacts strongly with bonded objects, companions, augmentations, platforms, or credentials,
- whether the Path modifies readiness, install, or capacity law indirectly.

---

## Path boundaries

This file should **not** silently absorb the following as if they were always Path-owned:
- all social capital,
- all starting resources,
- all access permissions,
- all augmentation or install states,
- all interface or resonance-like special channels,
- all bonded-object privileges,
- all licenses or credential authorities.

Those may interact with a Path. They are not automatically reducible to it.

---

## Cross-donor stance

After RHBF and Shadowrun, Astra's stable position is:
- Paths are real and important.
- Paths are not the only lawful build owner.
- A donor may pressure additional layers such as contacts, resources, access channels, augmentations, or special ratings.
- The starter Paths in this file are valid Astra exemplars, not a universal or exhaustive final roster.

This keeps Paths central without letting them monopolize the entire character architecture.
