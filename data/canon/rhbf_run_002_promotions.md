# RHBF Run 002 Promotions

Run ID: `rhbf_prod_run_002`  
Donor: `Ruthless Heavens, Boundless Fate`  
Purpose: `Record the RHBF-derived constructs accepted into Astra baseline grammar, accepted as Astra optional profiles, and explicitly not promoted from this donor pass.`

---

## 1. Integration rule applied

This ledger applies the standing RHBF integration rule:

**Promote the reusable grammar. Do not universalize RHBF’s exact defaults, exact numbers, exact ideology, exact catalog payloads, or exact donor-local naming unless Astra explicitly adopts them later.**

## 2. Accepted Astra baseline grammar

### Batch A — Player / chassis layer

- **Class → Path**
- **Skill → Technique** where the donor “skill” is an active codified power rather than a competency
- **Path as starting package rather than permanent exclusivity cage**
- **Domain-based access law**
- **Presence Pressure**
- **Lineage as inherited growth track**
- **Generalized technique-capacity doctrine**
- **Technique category split using Crown / Arm / Stride / Core as lawful capacity classes**
- **Technique Repository** as a lawful access/storage concept for learned techniques

### Batch B — Operational / world-interaction layer

- **hub → access → opportunity → return loop**
- **Nexus Hub** as a lawful hub-class concept
- **service-cluster hub design**
- **explicit access law**
- **Ascension Trial** as a lawful operational trial construct
- **Hunting Realm** as a lawful opportunity-space construct
- **Faction Contract** as a lawful faction-pressure construct
- **faction leverage / offer-denial logic**
- **Pressure Event** as a lawful world-pressure construct
- **trial procedure as operational grammar**
- **neutrality constraints for hubs** as a lawful hub-law pattern without requiring all hubs to be neutral

### Batch C — Content / schema layer

- **creature family as active mechanical classifier**
- **family posture grammar** at the grammar level rather than exact default posture bundles
- **primary-family / secondary-family hybrid slot**
- **stable actor / stat-block order** as schema law
- **threat-band field separated from donor-local scaling math**
- **resource-source ontology beyond slain actors**
- **typed harvest ecology**
- **site / environment extraction as lawful resource pattern**
- **harvest record fields**
- **natural treasure as progression-significant resource class**
- **yield tier grammar independent of exact donor payload names**
- **Bound Relic as owner-bonded growth relic class**
- **Bound Relic growth grammar through typed feeding**
- **Relic Trait** as a growth-bearing relic element
- **Resonant Feeding** as typed bound-relic input grammar
- **wondrous-item differentiation as a schema concern**

This RHBF pass confirms the following standing integration guidance:

- promote schema owner before content payload,
