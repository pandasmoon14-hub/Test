# Shadowrun Run 001 Promotions

Run ID: `shadowrun_prod_run_001`  
Donor: `Shadowrun Sixth World`  
Purpose: authoritative promotion ledger derived from the Shadowrun merge review and integration memo.

---

## Baseline grammar

The following Shadowrun-derived constructs are accepted as Astra baseline grammar.

### Batch A
- skills -> capabilities rather than pseudo-techniques
- explicit separation between operational capability, knowledge capability, and language capability
- layered package-stack character creation rather than one dominant chargen owner
- special access ratings distinct from primary attributes
- multi-route player-grade status including mundane, awakened, augmented, interface-native, and hybrid routes
- partial access states as lawful player-grade profiles
- augmentation as a lawful chassis-shaping route
- contacts as starting build pressure without giving Batch A ownership of wider social architecture
- starting resources as lawful build pressure without giving Batch A ownership of economy procedure
- interface-native versus gear-mediated technical access as distinct player states

### Batch B
- Brokered Contract as a lawful operation-loop grammar
- Legwork as a distinct pre-operation information phase
- a clear split between starting network layer and active access-node procedure
- relation grammar built on reach/leverage plus trust/commitment rather than one generic contact score
- Network Access Procedure as a real operational family without universal Matrix law
- Remote Platform Operations as a distinct procedure family
- layered infiltration as a reusable mixed-access operation pattern
- security ecology as a site-pressure model
- support tier / upkeep state as lawful recurring world pressure
- standing and heat as distinct consequence tracks
- situational advantage triggers as reusable leverage grammar without importing Edge economy

### Batch C
- installable augmentation as a primary item-family mode
- host-cost profile as the governing grammar for installed modifications
- host-target and install-class subfamilies
- rated technical asset grammar
- substrate profile for rated assets rather than one universal network metaphysic
- credential and authority-bearing asset grammar
- security-interface asset grammar
- platform schema distinct from both operator and procedure
- control-mode compatibility as a real object-side field
- multi-ontology actor support
- procurement and legality hooks for regulated assets
- explicit multi-ecology Batch C support so Astra can host manufactured, licensed, installed, salvaged, harvested, bonded, and mixed content ecologies together

## Optional profiles

The following Shadowrun pressures are accepted as lawful Astra optional profiles, but not universal defaults.

- `Interface rating` as the current best fix for the technomancer / Resonance collision
- trace-pressure / hostile-convergence clocks inspired by Overwatch-style pressure
- credential-heavy legal identity play as a recurring but non-universal world pattern
- accounting-heavy support-tier or upkeep simulation
- persistent network-enabled device profiles
- advanced implant-grade ladders
- attuned magical implements as one recurrent profile among several
- specific technical-access subtype splits that may or may not deserve later branching beneath Network Access Procedure
- single-scalar host-cost expression as one lawful implementation among several
- quality-band or implant-grade ladders where a donor supports them
- persistent connectivity profiles where a donor supports them

## Cross-donor significance

Shadowrun’s primary value is that it confirms Astra must remain broader than RHBF-shaped defaults. It strongly validates:

- technology-normal and technology-positive play without anti-tech baseline drift
- layered character-build grammar instead of one sovereign package model
- brokered and networked world-operation loops as lawful patterns rather than exotic exceptions
- manufactured, installed, licensed, and platform-based Batch C ecologies alongside organic and harvested ecologies
- access-state doctrine that distinguishes interface-native, gear-mediated, awakened, augmented, and hybrid states cleanly
