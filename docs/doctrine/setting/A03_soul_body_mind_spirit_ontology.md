# A03 Soul / Body / Mind / Spirit Ontology

## 1. Purpose and status
- File ID: A03
- Layer: 1_setting_spine
- Phase: 1A
- Status: draft
- Purpose: Define Astra doctrine-level ontology boundaries for body, soul, mind, and spirit; identity continuity; AI personhood; and persistence classifications without importing donor metaphysics as Astra default law.

## 2. What this file owns
- Doctrine-level component taxonomy for body, soul, mind, and spirit.
- Identity continuity classification grammar across embodied, augmented, copied, and transferred contexts.
- AI personhood classification boundaries at doctrine level.
- Possession/bonding ontology boundaries and non-mechanical classification handling.
- Death/persistence boundary classification outcomes at doctrine level.
- Source-local and quarantine posture for unresolved ontology pressure.

## 3. What this file must not own
- Specific actor stats.
- Specific implants or cyberware definitions.
- Specific resurrection mechanics.
- Afterlife planes.
- Gods.
- Factions.
- Planets.
- Planes.
- Resource pools, costs, recharge, or backlash.
- Action economy, damage, or conditions.
- Cultivation stages or progression.
- Runtime actor state or event commits.
- Accepted lexicon terms.
- Canon lore declarations.
- Final Core Rulebook prose.
- Donor metaphysics as Astra default law.

## 4. Required definitions
- **Body component**: the materially embodied substrate classification used for ontology mapping, without assigning numeric mechanics.
- **Soul component**: a continuity-bearing metaphysical component classification used where donor sources assert soul-like identity properties.
- **Mind component**: a cognition/mentality component classification covering memory, reasoning, and perspective carriers.
- **Spirit component**: a non-body, non-mind metaphysical agency component classification where sources assert spirit-like agency.
- **Identity continuity profile**: a doctrine-level determination of whether continuity claims are direct, transformed, forked, source-local, quarantined, or escalated.
- **AI personhood candidate**: an AI entity classification candidate eligible for ontology review without presuming equipment-only treatment.
- **Upload/copy instance**: a transfer or duplication result classification that must be tracked as continuity-relevant, not auto-resolved.
- **Possession/bonding construct**: a relationship classification between components/actors requiring ontology review boundaries.
- **Death/persistence boundary event class**: a non-runtime doctrine classification for persistence claims across transition boundaries.
- **Source-local ontology construct**: donor ontology retained with provenance and local scope only.
- **Quarantined ontology construct**: unresolved ontology held outside active Astra defaults pending later review.
- **Escalated ontology problem**: contradiction requiring formal downstream doctrine resolution.

## 5. Core doctrine rules
- A03 defines ontology classification grammar and boundaries only; it does not define tactical or runtime mechanics.
- A03 remains `A03_soul_body_mind_spirit_ontology.md` and is not renamed.
- K01 remains separate and lexicon governance is not moved into A03.
- A03 is doctrine draft guidance and is not current.
- Donor ontology can be mapped, retained source-local, quarantined, or escalated, but not auto-promoted as Astra default law.

## 6. Conversion mapping rules
- **Direct ontology mapping**: donor concept maps to an existing Astra ontology component class without semantic distortion.
- **Normalized ontology mapping**: donor concept maps after explicit normalization preserving provenance and boundary notes.
- **Forked continuity mapping**: copying/splitting outcomes are recorded as continuity-fork classifications rather than silently merged.
- **Source-local retained ontology construct**: donor concept retained for local conversion continuity without Astra-canon authority.
- **Quarantined ontology construct pending later doctrine**: unresolved ontology is isolated pending later doctrine/canon review.
- **Escalated ontology contradiction**: contradiction or unclassifiable construct is escalated for formal doctrine resolution.

## 7. Source-local handling
- Donor soul/body/mind/spirit constructs can remain source-local with provenance anchors.
- Donor-specific afterlife, resurrection, possession, or AI metaphysics remain source-local unless later doctrine explicitly adopts equivalents.
- Repeated donor pressure may produce canon candidates only through later K-layer governance.
- Unclassifiable constructs are quarantined or escalated rather than normalized by invention.
- Source-local handling preserves convertibility while preventing donor-law creep into Astra defaults.

## 8. Donor pressure absorbed
- Pressure to import donor soul laws as universal Astra truth.
- Pressure to treat AI strictly as equipment regardless of ontology evidence.
- Pressure to collapse copied/uploaded identities into single unquestioned continuity outcomes.
- Pressure to force donor possession/bonding metaphysics into global doctrine defaults.
- Pressure to equate extraction success with canon adoption.

## 9. Hard refusals / rejected imports
- D&D soul or afterlife mechanics are not Astra default ontology law.
- Shadowrun essence/astral assumptions are not Astra default ontology law.
- Cyberware is not presumed to be metaphysically neutral in all contexts by default.
- AI is not presumed to be non-person equipment by default.
- Contradictory continuity claims are not silently merged by invention.

## 10. Escalation triggers
- A proposed construct requires ontology component types outside current A03 grammar.
- Identity continuity classification contradicts A04/A05/A06 downstream constraints.
- Augmentation boundary handling contradicts A02 Source Field relation posture.
- A proposal attempts to move K01 lexicon governance into A03.
- A proposal requests runtime state/event ownership inside A03.
