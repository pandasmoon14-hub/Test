# SM08 Controlled Pilot Packet Placeholder Validation and No-Donor Safety Harness

## 1. Purpose and status

SM08 is a controlled pilot packet placeholder validation and no-donor safety harness file only. It adds SM08 as a controlled pilot packet placeholder validation and no-donor safety harness only.

SM08 does not run conversion. SM08 does not create actual converted donor content or pilot outputs. SM08 does not select or embed real donor content. SM08 does not create benchmark/evaluation/training corpora. SM08 does not create executable schemas, runtime contracts, final mechanics, canon/sourcebook/live-play/training content, or registry promotion.

SM08 validates SM07 placeholder scaffold fixtures. SM08 adds no-donor safety checks for placeholder/synthetic/non-donor labeling, capture-slot presence, obvious donor-content artifact markers, and sentinel donor-proper-noun markers.

SM08 preserves SM02 no-waiver posture and SM05 authorization boundary. SM08 states harness limitations and refuses treating no-donor sentinel checks as legal/IP clearance.

SM08 cannot waive SM02 gates and cannot bypass SM05 authorization.

Status posture:

- SM08 is validation/safety-harness infrastructure only, not actual conversion.
- SM08 validates that any SM07 scaffold fixtures are synthetic, non-donor, and clearly labeled.
- SM08 may use document-local validation labels, but those labels are not registry values.

## 5. No-donor safety checks definition

SM08 defines the following no-donor safety checks:

- placeholder/synthetic/non-donor labeling check: verifies that any scaffold fixture files contain explicit "placeholder", "synthetic", and "non-donor" labels.
- capture-slot presence check: verifies that required capture slots are present in scaffold fixtures.
- obvious donor-content artifact marker check: scans for known donor-content artifact markers such as "converted output:", "donor excerpt:", "statblock:", "source table:", "source map:".
- sentinel donor-proper-noun marker check: scans for known donor proper nouns that should never appear in Astra scaffold material.

Known donor markers include: Wizards of the Coast, Dungeons & Dragons, D&D, Forgotten Realms, Faerûn, Mind Flayer, Beholder, Displacer Beast, Githyanki, Githzerai, Illithid, Owlbear, Rust Monster, Umber Hulk, Bulette.

Donor content artifact markers include: converted output:, donor excerpt:, statblock:, source table:, source map:, donor statblock, donor table, donor map, donor setting prose, donor proper noun.

## 6. Placeholder/synthetic/non-donor labeling requirements

If scaffold fixture files exist, they must:

- be synthetic;
- be non-donor;
- be clearly labeled as placeholder;
- contain no real donor excerpts;
- contain no real donor statblocks;
- contain no donor tables;
- contain no donor maps;
- contain no donor setting prose;
- contain no donor proper nouns;
- contain no protected expression.

Placeholder fixture prose should use abstract terms such as synthetic packet, placeholder source, placeholder range, placeholder evidence reference, placeholder ledger slot, and placeholder review decision point. It must not contain real donor excerpts, donor statblocks, donor tables, donor maps, donor setting prose, donor proper nouns, protected expression, converted content, training data, canon, sourcebook prose, or runtime import data.
