SM11 is a real pilot authorization packet preparation-control file only. It defines controls for preparing a future real pilot authorization packet through metadata, external-reference posture, owner routing, legal/IP gating, no-waiver posture, and non-execution boundaries.

SM11 does not create a real authorization packet. SM11 does not make an actual authorization decision. SM11 does not authorize actual execution. SM11 does not approve execution. SM11 does not run conversion. SM11 does not create actual converted donor content or pilot outputs.

## 6. Real pilot authorization packet preparation definition

Real pilot authorization packet preparation means assembling controls for a later reviewer to draft a real packet without copying donor material into the repository. The preparation layer may describe what metadata, owners, references, routing decisions, legal/IP checks, and dependency evidence a future real packet must be able to show.

Preparation is not packet creation. Preparation is not source clearance. Preparation is not authorization. Preparation is not execution approval. Preparation is not conversion execution readiness. Preparation is not final mechanics, schema, runtime, canon, sourcebook, live-play, or training readiness.

## 7. Real packet versus dry packet boundary

The boundary is explicit:

- SM09 dry authorization packet template: placeholder/synthetic/non-donor template.
- SM10 dry authorization validation: reviewer gate for dry packets.
- SM11 preparation controls: controls for preparing a future real authorization packet.
- Future real authorization packet: not created by SM11.
- Future execution PR: not created by SM11 and still downstream.

A dry packet proves only that placeholder structure can be represented. SM11 preparation controls prove only that a later real packet can be prepared under stricter metadata, external-reference, owner-routing, and legal/IP posture. Neither dry packet nor SM11 preparation controls create execution approval.

## 8. Preparation eligibility prerequisites

Preparation cannot proceed unless the future packet can show all of the following without embedding donor content:

- SM02 gates have no waived minimum gate.
- SM03 dry-run posture is represented.
- SM04 rubric and handoff are represented.
- SM05 authorization slots are represented.
- SM06 output-capture slots are represented.
- SM07 scaffold/review-harness slots are represented.
- SM08 no-donor safety posture is represented.
- SM09 dry template slot structure is available.
- SM10 reviewer validation posture is available.
- Legal/IP owner is named.
- Future review owner is named.
- Repair/quarantine/failure-report paths are named.

If any prerequisite is missing, preparation must receive a blocked, repair, quarantine, legal/IP, pending_schema, or deferred-gap route rather than approval.

## 9. Legal/IP and external-reference preparation posture

SM11 defines a metadata-only / external-reference preparation posture. Real donor material must not be embedded in this PR. A future real packet may only identify donor source material through controlled metadata or external-reference posture if legal/IP routing allows it. Any external reference must be reviewed by a legal/IP owner before execution is proposed.

No raw donor excerpts, statblocks, tables, maps, setting prose, protected expression, or donor proper nouns may be copied into the repo as part of SM11. Metadata is not legal/IP clearance. External-reference readiness is not source clearance. No-donor sentinel tests are not legal/IP clearance.

A future real packet should record whether each external reference is allowed, restricted, pending legal/IP review, source-local only, or blocked. That record remains preparation evidence only and cannot become execution permission.

## 10. No-donor-in-repository preparation rule

SM11 enforces a no-donor-in-repository preparation rule. This PR and any SM11 preparation record must not place real donor material in repository files, tests, fixtures, examples, comments, schemas, ledgers, corpora, or generated outputs.

The rule covers donor excerpts, statblocks, tables, maps, setting prose, protected expression, proprietary names, copied mechanical blocks, converted donor content, pilot outputs, sourcebook-ready examples, benchmark corpora, evaluation corpora, training data, fine-tuning data, and model behavior data. Violations are hard failures routed to quarantine and legal/IP review.

## 11. SM02 no-waiver preparation requirement

SM11 preserves the SM02 no-waiver posture. A future real packet preparation record must state that SM02 minimum pilot readiness gates are present and no minimum gate is waived. Missing, ambiguous, waived, or ownerless SM02 evidence blocks preparation.

SM11 cannot waive SM02 gates. If SM02 evidence is incomplete, the only lawful preparation outcomes are repair, quarantine, deferred-gap routing, or refusal to draft the real packet.

## 12. SM03 dry-run preparation requirement
