## 6. Controlled pilot packet scaffold definition

A controlled pilot packet scaffold is repo-local placeholder/test structure for organizing future authorized pilot packet metadata and output-capture expectations. It is a planning scaffold for a later authorized packet, not the packet itself, not converted content, not a donor selection, not execution approval, not a schema implementation, and not runtime import data.

The scaffold may reserve placeholder locations for future packet metadata, evidence references, capture slots, ledgers, routing statements, review notes, and failure-report shapes. These reserved locations remain textual expectations only until a later owner creates an authorized execution packet under SM05 and SM06 controls.

## 7. Review harness definition

A review harness is repo-local placeholder/test structure for checking whether future pilot packet outputs can be routed to SM04 review, repair, quarantine, or lawful fallback. It is not an evaluator, not a benchmark corpus, not an evaluation corpus, not a final validator, not model behavior policy, not runtime behavior, and not canon acceptance.

The review harness may check whether required capture slots are present, whether SM05 authorization and SM06 output-capture posture are referenced, whether scope expansion is blocked, whether donor content leakage is blocked, whether SM04 review handoff is represented, and whether repair/quarantine/failure-report routing is visible.
