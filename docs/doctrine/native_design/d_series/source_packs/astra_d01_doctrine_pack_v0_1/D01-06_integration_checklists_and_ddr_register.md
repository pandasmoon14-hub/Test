# D01-06 Integration Checklists and DDR Register

## Purpose

This file records accepted D01 decisions and provides validation checklists.

## Accepted decision register

### D01-ATTR — Attribute Architecture

Accepted:
- D01 uses seven visible attributes plus hidden Pneuma.
- Familiar functional territory is deliberate, but donor math is not imported.
- Attributes are not skills, powers, or derived stats.

### D01-VISIBLE — Visible Attribute Set

Accepted:
- Vessel, Finesse, Reason, Insight, Integrity, Glamour, and Animus are visible attributes.
- Insight probes; Reason interprets.
- Integrity includes resilience and structural/identity coherence.
- Glamour includes social force and reality-adjacent presence.
- Animus covers inner/spiritual/cultivation force.

### D01-PNEUMA — Hidden Pneuma

Accepted:
- Pneuma is backend-only.
- Pneuma is not player-visible, not a spendable resource, and not an ordinary luck stat.
- Pneuma shifts rarely through significant cosmic/fate events.
- Pneuma can feed hidden PCR-like pressure.

### D01-LATTICE — Breakthrough Lattices

Accepted:
- Each visible attribute may have an independent breakthrough lattice.
- Pneuma has no ordinary breakthrough lattice.
- Breakthrough history and post-breakthrough altered state should be recordable.

### D01-DERIVED — Derived Pressure Handoffs

Accepted:
- D01 may identify derived pressure signals.
- D03/D07/etc. own formulas and consequences.
- Pneuma has no pool.

## D01 ownership checklist

D01 owns:
- attribute meanings;
- visible/hidden axis distinction;
- Animus/Pneuma boundary;
- attribute conversion candidates;
- breakthrough lattice substrate;
- derived pressure handoff signals.

D01 does not own:
- final derived-stat formulas;
- Power Economy pools;
- advancement gate procedure;
- skill lists or methods;
- route powers;
- harm mechanics;
- actor body mechanics;
- world consequences;
- live-play narration.

## Attribute validation checklist

For any attribute-adjacent construct, ask:

1. Is it broad capability, method, power, resource, harm, actor substrate, object-state, world-state, or source-local?
2. Which visible attribute, if any, does it map to?
3. Does it map to Animus or is it a separate D03/D06 resource/power construct?
4. Is luck/fate pressure being incorrectly mapped to Pneuma?
5. Does it import donor math?
6. Does it require D03, D04, D05, D06, D07, D08, D09, or D10 handoff?
7. Is source-local retention safer?

## Handoff matrix

| D01 pressure | Owner |
|---|---|

## Pre-generation risk queue

1. D01 imports donor ability-score math. Fix: map by function only.
2. Pneuma becomes player-visible luck. Fix: backend-only rule.
3. Animus absorbs every power system. Fix: route by function to D03/D06.
4. Attributes become skills. Fix: D05 owns methods.
5. Breakthrough lattices become level system. Fix: D04 owns advancement.
6. Integrity/Glamour lose clarity. Fix: preserve structural coherence and social/reality-facing presence.
7. Derived pools are finalized too early. Fix: hand off formulas to D03/D07/etc.
