## Purpose

D14 defines Astra doctrine for active movement through uncertain places and the lawful discovery of new state. It governs travel posture, route and area selection, navigation, scouting, mapping, discovery opportunities, map-state updates, site entry, watches, ambush exposure, travel pressure, source-local exploration systems, and donor exploration procedure mapping.

D14 is not a hex-crawl file, dungeon-turn file, random-encounter file, starship-travel file, planar-cosmology file, or map schema file. It is the owner-boundary and procedure layer that prevents those donor systems from becoming Astra defaults.

## Locked architecture

D14 uses the **Posture–Route–Discovery Architecture**:

```text
travel or exploration intent
  -> posture declaration
  -> route / area selection
  -> known-state review
  -> risk and PCR assessment
  -> movement or exploration interval
  -> discovery / hazard / encounter / transition check
  -> map-state update
  -> owner-file handoff
  -> continue, enter site, shift cadence, or end travel
```

## Core locks

Travel posture is first-class and is not a generic bonus. Routes are functional paths/vectors with knowledge states. D14 owns discovery procedure and classification; D10 owns actual world-state/information-state; D11 owns presentation. Party-facing map-state is distinct from actual truth. Environmental PCR is a travel/exploration pressure input. Donor travel systems map, source-local retain, quarantine, or escalate by function rather than label.
