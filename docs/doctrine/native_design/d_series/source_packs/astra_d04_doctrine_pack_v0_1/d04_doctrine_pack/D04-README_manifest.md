Generated from: Design Session Record 01 continuation through D04 closure  
Scope: advancement, proof, levels, tiers, awakening, breakthroughs, capstone, and runtime/schema handoff notes

## Purpose

This pack converts the accepted D04 design decisions into a multi-file doctrine set. It is intentionally split so D04 does not become a megafile. Each file owns a discrete advancement concern while preserving handoffs to D02, D03, D05, D06, D07, D08, D09, D10, Batch C schemas, canon review, and runtime.

D04 defines Astra's advancement architecture. It does not define final Expression formulas, final Power Economy math, final skill lists, final Path/Class/Dao feature tables, final harm/death math, final item/relic schemas, final world/faction systems, runtime implementation code, or sourcebook prose.

## Doctrine status rule

These files preserve accepted D04 design decisions. They are not canon sourcebook prose and do not directly implement runtime state. They should enter the repo as doctrine-draft files or be promoted to the project's preferred doctrine status through the same status/version procedure used for other Astra doctrine.

## Standing constraints preserved

D04 obeys these project laws:

1. Donor assumptions are evidence only.
2. Design doctrine is not canon by volume or polish.
3. Runtime state is not model memory.
4. Narration is not mechanical authority.
5. Training examples are not doctrine.
6. Conversion output is not canon until promoted.
7. Every donor construct still receives a lawful outcome in conversion intake.
8. Advancement systems must survive a large mixed donor corpus, including fantasy, science fiction, cultivation, class-based, point-buy, narrative/tag, cyberware, biotech, vehicle/platform, companion/summon, crafting, faction, settlement, adventure-path, and bestiary material.
