# D04-05 — Breakthrough Procedure and Payloads

Status: `doctrine-draft / accepted D04 design decisions`  
Layer: D04 advancement doctrine  
Owner: breakthrough architecture  
Depends on: D02 Resolution, D03 Power Economy and Expression Doctrine, D04 Proof Bundles, D07 Harm later integration

## 1. Purpose

This file defines how breakthroughs are identified, prepared, resolved, and recorded. A breakthrough is a transformative threshold event. It is not an ordinary level reward and not an arbitrary roll.

## 2. Core breakthrough principle

Proof earns the threshold. Risk determines the resolution method. Player response determines posture. Outcome determines payload and consequence.

Safe, stable, proof-supported breakthroughs usually avoid random rolls. Risky, forced, corrupted, overripe, misaligned, contested, or pushed breakthroughs use D02 outcome lanes, clocks, resistance checks, opposition procedures, dangerous push rules, or source-local procedures.

## 3. Breakthrough vectors

A breakthrough may affect visible attribute threshold lattice, Power Economy integration, Expression capacity/carrier/method, Path/Class/Dao route, skill/profession mastery, Vessel/body transformation, Integrity/identity coherence, Animus/internal force structure, external anchor, bond/companion, item/relic/tool, territory/domain/world-state, corruption/purification state, whole-character state, or capstone state.

## 4. Breakthrough threshold states

| State | Meaning |
|---|---|
| Stable | No active breakthrough pressure. |
| Pressured | Proof or strain is accumulating. |
| Eligible | Proof and conditions allow breakthrough attempt. |
| Delayed | Eligible but intentionally held. |
| Focused | Held and aimed at a declared vector. |
| Overripe | Held too long or pressure has intensified. |
| Blocked | Missing proof, structure, route, tier, anchor, or owner-file support. |
| Unstable | Breakthrough possible but risk is elevated. |
| Corrupted | Hostile/Vitia/tainted pressure affects threshold. |
| Coerced | External force is shaping threshold. |
| Forced | Threshold is being imposed or triggered. |
| Contested | Opposition actively resists or redirects threshold. |
| Lethal-risk | Crippling/lethal/unmaking outcome is explicitly on table. |
| Resolved | Breakthrough completed or closed. |
| Shattered | Failed severely; structure damaged. |

## 5. Breakthrough risk posture

| Risk posture | Meaning | Resolution tendency |
|---|---|---|
| Safe | Proof, route, capacity, timing align. | Formal or choice-gated. |
| Strained | Some pressure/cost exists, manageable. | Choice-gated or minor test. |
| Unstable | Route, carrier, economy, proof has instability. | Project/clock or d20 test. |
| Dangerous | Serious harm, corruption, shatter, route damage possible. | Warning + d20/project. |
| Catastrophic | Crippling/lethal/unmaking risk exists. | Explicit warning + dangerous push/forced procedure. |

Lethal breakthrough outcomes require lethal-risk posture, catastrophic posture, or established pressure trail.

## 6. Breakthrough Resolution Stack

Every breakthrough passes through this stack:

1. Identify Vector
2. Check Proof Bundle
3. Check Threshold State
4. Determine Risk Posture
5. Present Warning if required
6. Player Response
7. Choose Resolution Mode
8. Apply Outcome
9. Apply Payload
10. Commit State Deltas

## 7. Resolution modes

### 7.1 Formal Breakthrough

Use when the breakthrough is stable, aligned, proof-supported, and not under meaningful danger. No d20 roll required. Confirm proof, present legal payload choices if applicable, apply expected cost if any, and commit state delta.

### 7.2 Choice-Gated Breakthrough

Use when the breakthrough is eligible but has multiple lawful outcomes. No random roll unless the player chooses risky option.

### 7.3 Project / Clock Breakthrough

Use when breakthrough requires extended effort, such as stabilizing a dangerous economy, re-authoring an Expression, preparing a whole-character breakthrough, purifying corruption, building a tool-anchor, training for route fusion, or repairing a carrier before proceeding.

### 7.4 D20 Threshold Test

Use when breakthrough is immediate and uncertain: unsafe, sufficient but unstable proof, overripe threshold, pushed beyond preparation, strained carrier, misaligned route, volatile economy, corruption, or crisis. D02 owns the roll. D04 owns breakthrough interpretation.

### 7.5 Contested / Coerced Breakthrough

Use when another force is actively shaping the breakthrough: enemy ritual, faction procedure, pact seizure, corruption redirect, relic imposition, rival block, or domain rejection.

### 7.6 Dangerous Push Breakthrough

Use when the player knowingly attempts breakthrough beyond safe authorization. Requires explicit warning, risk category, possible crippling/lethal outcomes if applicable, safer alternatives, player confirmation, and D02 test or crisis procedure.

## 10. Payload categories

Breakthrough payloads can touch more than one category. A payload may contain benefit, consequence, state transformation, scar, access, and world recognition simultaneously.

| Payload category | Meaning |
|---|---|
| Capacity Payload | More capacity, tolerance, reservoir, carrier, recovery, Endura, partition. |
| Access Payload | New route, method, economy, axis, teacher/faction rite, domain access. |
| Expression Payload | Expression stabilization, re-authoring, scope opening, carrier relation, method shift. |
| Route Payload | Path/Class/Dao formalization, branch, drift, fusion, severance, mutation, clarification. |
| Attribute Payload | Visible attribute threshold shift, post-breakthrough altered state. |
| Economy Payload | Power Economy integration, partition, confluence, overdraw tolerance, rejection, reconstitution. |
| Carrier / Slot Payload | Slot opening, carrier repair, carrier expansion, externalized burden. |
| Resistance / Recovery Payload | Harm resistance, corruption resistance, scar integration, recovery improvement. |
| Title / Achievement Payload | Formal proof binding into title, achievement, recognition. |
| Bond / Anchor Payload | Companion, relic, tool, platform, domain, territory, faction, oath, or external anchor deepens. |
| World-State Payload | Faction/world/domain/territory reacts or changes. |
| Scar / Cost Payload | Known limitation, debt, vulnerability, scar, oath, memory loss, relationship cost. |
| Corruption / Purification Payload | Vitia trace, hostile pressure, purification progress, corruption acceptance. |
| Actor-State Payload | Body/spirit/AI/swarm/platform/whole-character transformation. |
| Capstone Payload | Final-form, world/plane pressure, mythic recognition, Ascension Access relevance. |

## 11. Payload quality bands

| Band | Meaning |
|---|---|
| Null | No payload; breakthrough failed or blocked. |
| Residual | A trace remains; may become proof or project. |
| Reduced | Lesser lawful manifestation than intended. |
| Partial | Some intended change occurs but remains incomplete. |
| Fragile | Works but unstable, narrow, temporary, or costly. |
| Scarred | Works through a scar, debt, limitation, corruption trace, or wound. |
| Clean | Expected payload with expected cost. |
| Reinforced | Payload is stable, durable, or easier to develop. |
| Enhanced | Payload gains secondary lawful benefit or improved quality. |
| Rare | Stronger/specific/unique payload from high contextual proof. |
| Defining | Character-shaping payload that changes route, identity, or world relation. |
| Mythic | World-facing or campaign-shaping payload. |
| Capstone | Endgame final-form payload. |

Payload quality must respect proof, tier, route, cost, and risk posture.

## 12. Failure payloads

Breakthrough failure can have payload-like consequences: threshold remains unresolved, proof becomes corrupted or badly bound, route becomes unstable or severs, carrier strains or burns, economy rejects actor, Expression backlashes, Integrity fractures, Vessel ruptures, actor-state shifts unintentionally, external anchor damages, world/faction/domain backlash, forced mutation, crippling state, or lethal outcome if risk posture permits.

## 13. D02-to-D04 mapping

Safe breakthroughs usually avoid rolls. When a breakthrough uses D02, map D02 lanes into D04 outcomes.

| D02 lane | Trigger | D04 breakthrough result |
|---|---:|---|
| Critical Success | Natural 20, if action vector lawful | Best lawful breakthrough result available; may amplify, stabilize, reduce cost, reveal rare lawful option, soften danger. |
| Exceptional Success | Margin +5 or higher | Enhanced breakthrough; desired result succeeds with improved payload or secondary lawful benefit. |
| Clean Success | Margin 0 to +4 | Clean breakthrough; expected payload and declared cost apply. |
| Partial Result | Margin -1 | Partial breakthrough without backlash; incomplete progress, no added complication beyond declared cost. |
| Mixed Result | Margin -2 to -3 | Partial breakthrough with complication; scar, instability, drift, cost, strain, or reduced payload. |
| Failure with Consequence | Margin -4 to -7 | Desired breakthrough fails; consequence lands; no guaranteed useful return. |
| Severe Failure | Margin -8 or worse | Severe failure; may shatter, corrupt, cripple, invert, burn, or damage structures depending on posture. |
| Critical Failure | Natural 1 | Worst lawful failure within declared risk posture; catastrophic only if lethal/catastrophic risk was already on table. |

## 14. Lawful attempt vector rule

A breakthrough roll requires a lawful attempt vector. Valid vectors include sufficient Proof Bundle, partial proof plus dangerous push, route access, relevant attribute threshold, Power Economy contact, Expression/carrier basis, external anchor, teacher/faction/ritual support, forced/coerced pressure, source-local mechanism, or breakthrough crisis.

Without a vector, the request becomes a project, research question, training goal, route-seeking objective, mundane action, dangerous self-harm attempt, or source-local request.

## 15. Natural 20 rule

A natural 20 gives the best lawful result available. It cannot bypass missing tier authorization, missing route, missing proof, missing carrier, missing access, or missing action vector. If the desired breakthrough is not lawful, a natural 20 may produce discovery, route seed, project, lesser lawful manifestation, avoided backlash, or rare warning. It cannot grant unsupported final power.

## 16. Natural 1 rule

A natural 1 produces the worst lawful failure within declared risk posture. It does not cause arbitrary death. Death, crippling, unmaking, or actor-state collapse require lethal-risk posture, catastrophic posture, source-local lethal procedure, or established pressure trail.

## 17. State deltas by outcome

| Result band | Minimum state delta |
|---|---|
| Critical/Exceptional Success | Payload applied, proof bound/transformed, threshold resolved, possible secondary benefit. |
| Clean Success | Payload applied, cost committed, threshold resolved. |
| Partial Result | Partial payload or progress state, cost committed, threshold remains partly unresolved. |
| Mixed Result | Partial payload plus complication, scar, instability, route drift, or cost. |
| Failure with Consequence | Cost/consequence committed, no desired payload, threshold/route/proof state changes. |
| Severe Failure | Harm, corruption, shatter, carrier/route/economy damage, or major failure state. |
| Critical Failure | Worst lawful failure within posture, catastrophic only if posture permits. |

## 18. Crippling and lethal breakthrough rule

Breakthroughs can be crippling or lethal, but only under appropriate posture and pressure. Crippling and lethal outcomes may occur when the player selected Push after warning; posture is lethal-risk or catastrophic; forced/coerced fiction already carried known danger; source-local procedure clearly includes lethal risk; capstone or Ascension attempt carries declared catastrophic pressure; or severe/critical failure occurs inside a declared lethal-risk event.

D07 later owns exact injury/death/unmaking mechanics.

## 19. Safe breakthrough protection

A safe, stable, proof-supported breakthrough should not be randomly invalidated by unnecessary rolling. If there is no meaningful uncertainty, danger, opposition, instability, haste, corruption, misalignment, or unsafe push, use Formal or Choice-Gated resolution.

## 20. Validation rules

A breakthrough procedure is invalid if it rolls for every breakthrough by default; makes proof irrelevant; kills characters without lethal-risk posture or pressure trail; grants impossible results on natural 20; lets natural 1 ignore posture limits; gives useful return on every failure; lacks player response options for dangerous breakthroughs; lacks state deltas; bypasses D03/D06/D07/D08/D09/D10 owner validation; or treats breakthrough payload as a normal level reward.
