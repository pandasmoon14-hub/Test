# A00 Mechanical Posture and Ruleset Non-Adoption

## Status
- File ID: A00
- Layer: 0_control
- Phase: 0
- Status: draft

## Decision
Astra doctrine remains a control/governance frame at this stage and does **not** adopt donor rulesets, donor mechanics, or live-play procedures as executable Astra rules.

## Mechanical posture
- Donor material may be extracted, classified, and routed.
- Donor mechanics may be mapped to doctrine-ready placeholders.
- Donor mechanics are not auto-promoted into Astra mechanical canon.
- Conversion permission is not mechanical adoption.

## Non-adoption ruleset guardrails
- No donor rules text becomes Astra rule authority by default.
- No donor subsystem names become required Astra subsystem names by default.
- No donor balancing assumptions are treated as Astra balancing doctrine without explicit downstream approval.

## Ownership boundaries
This control file owns:
- governance posture for mechanical non-adoption during doctrine bootstrap
- anti-creep constraints for donor mechanics import pressure

This control file must not own:
- core cosmology and dimensional architecture content (A01)
- lexicon governance and reserved terms policy (K01)
- downstream implementation/runtime behavior

## Escalation triggers
Escalate if any draft/current doctrine file:
- treats donor rules text as default Astra authority
- redefines A01 or K01 ownership boundaries
- bypasses registry tracking for control-layer exceptions
