## 10. Dependency relationships

Generated content may begin as an ephemeral proposal. Durable recurrence requires backend-owned schema, provenance, validation, event/state commitment, persistence, and reviewer-controlled eligibility. Canon promotion is separate from generated-content persistence. Source-local/generated/canon status must remain distinguishable.

Durable recurrence also remains downstream of RT-001 command/event commitment, RT-002 resource/consequence math, RT-003 combat/hazard/damage/recovery consequences, RT-005 context-packet/hidden-information projection, RT-011 validation/readiness tooling, and later RT-006/RT-007/RT-009 owner files where missions, rewards, clues, relationships, faction knowledge, RNG, tables, or oracle outputs are involved.

## 11. LLM non-authority rules

The LLM is prohibited from:

- creating durable generated records;
- deciding recurrence eligibility;
- assigning persistent IDs as authority;
- creating provenance records;
- writing generated content to files;
- promoting generated content to canon;
- treating generated narration as persisted setting truth;
- creating NPC/faction/location/item/mission/hazard/table records as backend truth;
- bypassing validation/reviewer approval;
- using model memory as generated-content storage.

The LLM may draft an ephemeral proposal only when a future backend/reviewer-controlled workflow asks for one. It may narrate or summarize generated material only as an explicitly labeled proposal or as backend-approved visible fact supplied by an authorized context packet.

## 12. Context-packet and narration handoff expectations

Context packets must distinguish proposed/generated/source-local/canon status before generated material reaches narration. Narration must not convert a generated candidate into persisted setting truth, must not imply recurrence eligibility, must not assign durable identity, and must not treat repeated mention as persistence. If a generated candidate is rejected, quarantined, or pending review, context packets must preserve that state rather than presenting it as canon or backend truth.
