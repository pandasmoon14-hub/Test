# Cross-Donor Consolidation Pass v0.1

Purpose: apply the RHBF + Shadowrun cross-donor promotions to Astra's distillation layer **without** widening doctrine or quietly canonizing donor-local payload.

This pass is intentionally narrow. It does not rewrite Batch A/B/C from scratch. It only updates the Astra-facing distillation layer so it reflects what survived both donors.

---

## 1. Consolidation law

Use this rule for every update in this pass:

- Promote **shared grammar**.
- Keep **optional profiles** explicitly optional.
- Keep **donor-local payload** out of Astra baseline files.
- Leave **unresolved mappings** named and provisional rather than silently deciding them.

This means:

- RHBF-specific topology, reserve semantics, anti-tech ideology, exact posture bundles, and exact relic math do **not** become baseline.
- Shadowrun-specific priority tables, Edge menu, Matrix cosmology, exact Essence law, and exact device/credential catalogs do **not** become baseline.
- Shared structural lessons **do** become baseline.

---

## 2. What is promoted by this pass

### Promote now as Astra baseline grammar

- layered build grammar rather than one universal chargen package model
- access law as real doctrine
- world-operation loop grammar
- contract / opportunity structures as lawful world procedure
- faction structure and leverage logic
- category / family / classification doing real mechanical work
- content schema broad enough for both organic and manufactured ecologies
- resource-source ontology broader than slain actors only
- bonded-growth object grammar

### Promote as Astra optional profiles

- tri-axis advancement / breakthrough gating
- Transit Seal style access instruments
- brokered-contract operation loop
- optional base-operations overlay
- owner-bonded relic / weapon growth
- augmentation host-cost grammar
- standing / heat / notoriety style consequence layers as separate but related profiles

### Do not promote in this pass

- RHBF exact `1/4/1/3` technique profile
- RHBF exact reserve-triplet semantics
- RHBF exact family posture bundles
- RHBF anti-tech hostility
- RHBF exact Stronghold Phase cadence
- Shadowrun exact priority table
- Shadowrun exact Edge menu
- Shadowrun exact Matrix cosmology
- Shadowrun exact Essence law
- donor-specific faction identities, proper nouns, and exact catalog math

## 6. Output expectation for this pass

After this consolidation pass, Astra should visibly reflect these truths:

- it supports layered character builds rather than one donor chain
- it supports multiple lawful access models rather than one token model
- it supports brokered and non-brokered operation loops
- it supports both organic and manufactured Batch C ecologies
- it keeps family/category mechanically meaningful without canonizing one donor's exact defaults
- it can host both bonded relic growth and augmentation/platform grammar without collapse

If those outcomes are visible, the pass succeeded.

---

## 7. What this pass does **not** do

This pass does **not**:

- settle final reserve taxonomy
- unify Fate and Edge or any other reserve/risk economies
- canonize exact Essence law or exact RHBF reserve semantics
- choose donor three's final promotions in advance
- replace the full doctrine stack

Those remain for later pressure or later canon consolidation.
