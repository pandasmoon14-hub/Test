# Stage 9 — RHBF Batch C Execution Pack v0.1

## Scope

This pack operationalizes RHBF Parts 8–9 for AstraCloud.

Batch C covers:
- creature taxonomy
- stat block preservation and conversion rules
- evolving / ascendant beasts
- harvest and component extraction
- natural treasures
- bound relic and living-weapon progression
- gear and treasure support hooks

## Donor conclusions

RHBF Parts 8–9 provide a full combat-ecology loop, not a disposable appendix.
The donor text repeatedly links:
- creature family
- default defenses and special interactions
- stat block structure
- trophy / component / harvest output
- natural treasure economy
- living weapon trait growth

This is precisely the layer Astra needs for a reusable creature-and-gear framework.

## Astra conversion laws for Batch C

1. Creature families must do mechanical work.
2. Stat blocks must preserve structure before flavor.
3. Harvest outputs must be typed by creature family.
4. Natural treasures must remain progression-significant.
5. Living weapons convert to Bound Relics, not generic magic items.
6. Bound Relics must grow through Resonant Feeding, not arbitrary enchanting.

## Core Batch C conversions

- Living Weapon -> Bound Relic
- Living Weapon Traits -> Relic Traits
- natural treasure -> natural treasure (preserve concept, Astra-name case-by-case)
- beast core -> beast core / essence core (provisional)
- evolved beast -> ascendant beast
- humanoid form breakthrough -> anthromorphic breakthrough
- anti-magic coating and similar typed materials -> family-typed component inputs

## Implementation targets

This pack should be installed into AstraCloud and used together with the earlier Batch A and Batch B packs.

Batch C is the first donor pack that should strongly influence:
- bestiary conversion rules
- validator structure for stat blocks
- harvest output templates
- bound relic progression files
