# Cross-Donor Consolidation Pass v0.3

Purpose: update the Astra-facing consolidation plan after comparison of the first four completed donors:

- RHBF
- Shadowrun
- Lancer
- Starfinder

This file supersedes the earlier consolidation posture where it conflicts.

It is not a doctrine rewrite.  
It is an execution document identifying what should now be changed in the live Astra distillation layer, what should be added, what should be reframed, and what should still remain provisional.

---

## 1. Executive summary

The first four donors now require Astra to consolidate around six non-negotiable truths:

1. Astra must support **layered build grammar** rather than one sovereign player package.
2. Astra must preserve **theme/background-like identity** as a real build owner rather than flavor-only text.
3. Astra must preserve **operator / platform separation** as real baseline doctrine.
4. Astra must preserve **multiple operation cadences** without treating any one donor loop as universal.
5. Astra Batch C must explicitly support **multiple content ecologies**:
   - harvest / extraction,
   - manufactured / installable,
   - licensed / requisitioned,
   - salvage / refit,
   - bonded / attuned,
   - technological,
   - magical,
   - hybrid,
   - and mixed ecologies.
6. Astra must now treat **starships and broader vehicle/platform families** as core doctrine rather than later edge cases.

The distillation layer must reflect these truths in the live Astra-facing files, not only in donor-run notes and ledgers.

---

## 2. Consolidation stance

Use the following promotion trichotomy while applying changes:

- **Astra baseline grammar** = write directly into live distillation files
- **Optional recurring profile** = preserve as lawful support without presenting it as universal law
- **Donor-local retained structure** = keep out of live baseline files unless explicitly used as a bounded example
- **Still provisional** = mention only when needed as an unresolved watchpoint
