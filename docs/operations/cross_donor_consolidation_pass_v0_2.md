# Cross-Donor Consolidation Pass v0.2

Purpose: update the Astra-facing consolidation plan after comparison of the first three completed donors:

- RHBF
- Shadowrun
- Lancer

This file supersedes the earlier two-donor consolidation posture where it conflicts.

It is not a doctrine rewrite.  
It is an execution document that identifies what should now be changed in the live Astra distillation layer, what should be added, what should be reframed, and what should still remain provisional.

---

## 1. Executive summary

The first three donors now require Astra to consolidate around four non-negotiable truths:

1. Astra must support **layered build grammar** rather than one sovereign player package.
2. Astra must preserve **operator / platform separation** as real baseline doctrine.
3. Astra must preserve **multiple operation cadences** without treating any one donor loop as universal.
4. Astra Batch C must explicitly support **multiple content ecologies**:
   - harvest / extraction,
   - manufactured / installable,
   - licensed / requisitioned,
   - salvage / refit,
   - bonded / attuned,
   - and mixed ecologies.

The distillation layer must now be updated so that these truths are visible in the live Astra-facing files, not only in donor-run notes and ledgers.

---

## 2. Consolidation stance

Use the following promotion trichotomy while applying changes:

- **Astra baseline grammar** = write directly into live distillation files
- **Optional recurring profile** = preserve as lawful support without presenting it as universal law
- **Donor-local retained structure** = keep out of live baseline files unless explicitly used as a bounded example
- **Still provisional** = mention only when needed as an unresolved watchpoint
