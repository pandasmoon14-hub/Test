# Stage 10 — RHBF Integration and Merge Pack v0.1

This stage consolidates the RHBF donor work from Batches A, B, and C into one AstraCloud-ready merge layer.

## Purpose

The earlier packs established three donor bands:

- Batch A: core chassis, Paths, attributes, resistances, techniques, damage, crits
- Batch B: hub logic, travel gating, Trials, Hunting Realms, factions, Stronghold Phase
- Batch C: creatures, stat blocks, harvests, natural treasures, Bound Relics

This merge pack turns those separate threads into one coherent donor stream for the first real AstraCloud conversion run.

## What is merged here

- donor profile assumptions
- routing and chunk-family doctrine
- validator focus areas
- canonical-ready lexicon terms
- provisional lexicon terms
- canon candidates
- open adjudications
- first recommended run order

## Practical result

After installing this pack, AstraCloud should have enough merged RHBF doctrine to support a first serious conversion workflow:
1. convert chassis chunks first
2. convert activity/world chunks second
3. convert creature/gear chunks third
4. emit lexicon deltas and canon packets throughout
5. review open adjudications before promoting unstable terms into canonical doctrine
