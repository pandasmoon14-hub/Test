# Cross-Donor Promotions v0.1

Scope: RHBF + Shadowrun  
Purpose: Record what Astra should promote after surviving two contrasting donor passes.

This file is a cross-donor promotion ledger, not a donor-run artifact. It records the Astra-facing outcomes that survived comparison between RHBF and Shadowrun.

---

## 1. Promote now — Astra baseline grammar

These constructs are supported strongly enough across RHBF and Shadowrun to promote into Astra-facing baseline grammar now.

### Batch A

- **Layered build grammar**  
  Astra should support stacked character-build layers rather than assuming one universal package model.

- **Access law as real doctrine**  
  Permissions, gates, and special access structures must remain explicit and mechanically meaningful.

- **Capability differentiation without monoculture collapse**  
  Astra must support broad capabilities, specialized channels, and different access profiles without forcing all donor structures into one Path-and-Technique monoculture.

### Batch B

- **World-operation loop grammar**  
  Astra should support repeatable operation loops linking opportunity, access, execution, return, and consequence.

- **Contract / opportunity structure as lawful procedure**  
  Jobs, offers, contracts, assignments, and opportunity hooks are valid operational structures.

- **Faction structure and leverage logic**  
  Factions should function as active pressure engines, not only lore bodies.

- **Consequences that persist across operations**  
  Standing, pressure, fallout, attention, and related consequence layers are lawful recurring operational pressures.

### Batch C

- **Broad content-schema support for both organic and manufactured ecologies**  
  Astra must support creatures, harvests, relics, augmentations, devices, platforms, credentials, and other schema-bearing content without privileging one ecology.

- **Classification doing real mechanical work**  
  Families, categories, classes, or equivalent schema-level classifications must matter mechanically.

- **Resource-source ontology broader than slain actors**  
  Valuable outputs may come from actors, environments, sites, anomalies, assets, platforms, or mixed sources.

- **Bonded-growth object grammar**  
  Astra should support owner-linked, evolving, or growth-bearing objects as a lawful content grammar.

## 2. Promote as optional profiles

These constructs are useful and lawful, but should remain optional profiles rather than universal Astra baseline.

### Batch A optional profiles

- **Tri-axis advancement / breakthrough gating**  
  Strong pressure from RHBF, but not yet broad enough to universalize.

- **Special reserve / risk economies as separate profile tools**  
  Reserve-style or special-build economies are valid, but not yet one shared baseline.

- **Contacts and resources as meaningful Batch A build pressure**  
  Strongly reinforced by Shadowrun, but should remain configurable by donor and campaign mode.

### Batch B optional profiles

- **Transit-seal style access instruments**  
  Explicit access instruments are lawful, but not universal.

- **Brokered-contract operation loop**  
  A strong recurring pattern, but one major option among several valid operation structures.

- **Optional base-operations overlay**  
  Return-to-base / stronghold / support-cycle play is valid, but not mandatory cadence.

- **Separate standing / heat / notoriety style consequence layers**  
  Distinct social-operational consequence tracks are useful, but should remain profile-level unless later corpus pressure says otherwise.

- **Remote / proxy / platform operation**  
  A real procedural pattern, but not universal baseline for all donors.

### Batch C optional profiles

- **Weapon-shaped bonded-growth objects**  
  Bound Relics and equivalent growth-bearing items are strong, but not the only object-growth form Astra should support.

- **Augmentation host-cost grammar**  
  Installed augmentations with host burden, compatibility, or slot pressure are valid, but should stay broader than one donor’s exact law.

- **Credential and access assets as schema-bearing content**  
  Useful and important, but should remain one profile within the larger asset ecology.

- **Rated devices and operator-linked platforms**  
  Strongly supported by Shadowrun, but still profile-level until broader donor pressure arrives.
