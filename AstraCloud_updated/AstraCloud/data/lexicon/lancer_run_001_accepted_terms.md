## Terms intentionally not promoted

- exact `LL` as Astra progression law
- exact `HULL / AGILITY / SYSTEMS / ENGINEERING` labels as Astra attributes
- exact manufacturer names
- exact sitrep names
- exact frame names and statistical packages
- exact NHP / cascade vocabulary as Astra baseline metaphysics
- any wording that collapses pilot/operator and platform into one build owner
- any wording that turns triggers into Techniques by default
- any wording that collapses mounts and install capacity into one slot family

## Sensitive terms to keep under review

### platform-tuning matrix
Strong donor pressure, but still profile-level rather than baseline law.

### structure-stress durability profile
Likely useful across later donors, but still needs wider pressure before canon-grade universality.

### platform intelligence profile
Useful recurring family, but easy to overfit into one donor’s metaphysics.

### reserve declaration
Strong operation-support grammar, but still cadence-sensitive.
